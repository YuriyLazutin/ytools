ytools-query-file-name

This tool will show a dialog in which you can specify a file name, or you can use a standard "Open File"-dialog to select a file. This file name will returned back to requesting program. Also you can specify a message text to clarify you request for user. All parameters should be provided via command line.

Options:

ShortLongDescription
-m--messageA text message, it should clarify you request
-f--file-maskA filter mask witch will used during file selection. For example you can allow to select only *.dat files
-d--directoryA base directory. In this directory initially will open query file dialog
-t--dtitleA text string which will show in dialog header
-b--btntextThis text will show on Accept button, i.e. you can rename this button
-h--helpWill show help and exit

Examples:

Query any file

#!/bin/bash
# some actions here
$(ytools-query-file-name --message "Please select a source file name")
As a result following window will appear:



Query an image file

Files can be png, bmp, jpg or gif format. File formats are divided on 2 groups - lossless and lossy.

#!/bin/bash
# some actions here
$(ytools-query-file-name -m "Please select an image file" -f "*.png *.bmp ;; *.jpg *.gif")
As a result following window will appear:



Query an jar archive

Accept button will renamed to "Patch this!".

#!/bin/bash
# some actions here
$(ytools-query-file-name -m "Please select a bogous file" -f "*.jar" --btntext "Patch this!")
As a result following window will appear:



Query any file

But if you will use query file dialog it will opened from /opt/share directory.

#!/bin/bash
# some actions here
$(ytools-query-file-name --message "Please select a source file name" --directory "/opt/share")
As a result following window will appear:
